This guide covers how to:
* Filter events while sending them to the specified destinations.
* Filter destinations where the event data should be sent.

## Filtering events

When sending events to a destination via the **web device mode**, you can explicitly specify which events should be discarded or allowed to flow through - by allowlisting or denylisting them using the <a href="https://rudderstack.com/docs/stream-sources/rudderstack-sdk-integration-guides/event-filtering/">Client-side Event Filtering</a> feature.

## Filtering destinations

You can send the event data only to the selective destinations by passing an [integrations](#integrationopts)  object in the `options` parameter. RudderStack then loads or sends events only to the specified and enabled destinations.

* A sample snippet to send event data only to the **Google Analytics** and **Intercom** destinations:

```javascript
rudderanalytics.load(WRITE_KEY, DATA_PLANE_URL, {
  integrations: {
    All: false,
    "Google Analytics": true,
    Intercom: true
  }
});
```

* A sample `track` method to send event data only to the **Google Analytics** destination:

```javascript
rudderanalytics.track(
  "test track event GA3",
  {
    revenue: 30,
    currency: "USD",
    user_actual_id: 12345
  },
  {
    integrations: {
      All: false,
      "Google Analytics": true
    }
  }
);
```
