This guide lists the browser versions supported by JavaScript SDK, steps to install and use the SDK to identify your website users and track their actions.

## Supported browsers and versions

The JavaScript SDK is supported on the following browsers and their corresponding versions:  

| Browser         | Supported Versions |
| :------------------ | :--------------------- |
| Safari              | v7 or later            |
| IE                  | v10 or later           |
| Edge                | v80 or later           |
| Mozilla Firefox     | v47 or later           |
| Chrome              | v54 or later           |
| Opera               | v43 or later           |


<div class="infoBlock">
You can try adding the browser polyfills to your application if the SDK does not work on the browser versions you are targeting.
</div>

## Step 1: Install JavaScript SDK 

### Using CDN

To integrate the SDK with your website and load it asynchronously, add the following code in the `<head>` section of your website.

```javascript
<script type="text/javascript">
!function(){var e=window.rudderanalytics=window.rudderanalytics||[];e.methods=["load","page","track","identify","alias","group","ready","reset","getAnonymousId","setAnonymousId"],e.factory=function(t){return function(){var r=Array.prototype.slice.call(arguments);return r.unshift(t),e.push(r),e}};for(var t=0;t<e.methods.length;t++){var r=e.methods[t];e[r]=e.factory(r)}e.loadJS=function(e,t){var r=document.createElement("script");r.type="text/javascript",r.async=!0,r.src="https://cdn.rudderlabs.com/v1.1/rudder-analytics.min.js";var a=document.getElementsByTagName("script")[0];a.parentNode.insertBefore(r,a)},e.loadJS(),
e.load(<WRITE_KEY>,<DATA_PLANE_URL>),
e.page()}();
</script>
```

<div class="infoBlock">
Replace the <Link to="/resources/glossary/#write-key">Write key</Link> and <Link to="/dashboard-guides/overview/#data-plane-url">Data plane URL</Link> in the above snippet with actual values.
</div>

This code snippet:

- Creates an array to store the events until the `analytics` object is ready.
- Stores the following methods to replay them when the `analytics` object is ready:

| Method           | Description                                          |
| :--------------- | :--------------------------------------------------- |
| <Link to="/sources/sdks/rudderstack-javascript-sdk/load-js-sdk/#loading-javascript-sdk">`load()`</Link>     | Loads `analytics.js` with the specified write key. |
| <Link to="/sources/sdks/rudderstack-javascript-sdk/supported-api/#page">`page()`</Link>     | Records page views whenever a user visits a page.    |
| <Link to="/sources/sdks/rudderstack-javascript-sdk/supported-api/#track">`track()`</Link>   | Tracks user actions and other events.            |
| <Link to="/sources/sdks/rudderstack-javascript-sdk/supported-api/#identify">`identify()`</Link> | Associates users and their traits or actions.        |
|<Link to="/sources/sdks/rudderstack-javascript-sdk/supported-api/#reset">`reset()`</Link>    | Resets the `userid` and the associated traits.       |

- Loads the analytics object with the specified write key.
- Makes the `page()`call to track the page views. It auto-captures properties such as `path`, `referrer`, `search`, `title`, and `URL`. Refer to the <Link to="/event-spec/standard-events/page/">`page`</Link> method to override these properties. 

#### Minified code

```javascript
<script>
rudderanalytics=window.rudderanalytics=[];for(var methods=["load","page","track","identify","alias","group","ready","reset","getAnonymousId","setAnonymousId"],i=0;i<methods.length;i++){var method=methods[i];rudderanalytics[method]=function(a){return function(){rudderanalytics.push([a].concat(Array.prototype.slice.call(arguments)))}}(method)}rudderanalytics.load(<WRITE_KEY>,<DATA_PLANE_URL>),rudderanalytics.page();
</script>

<script src="https://cdn.rudderlabs.com/v1.1/rudder-analytics.min.js"></script>
```

#### Non-minified code

```javascript
<script>
  rudderanalytics = window.rudderanalytics = [];
  var methods = [
    "load",
    "page",
    "track",
    "identify",
    "alias",
    "group",
    "ready",
    "reset",
    "getAnonymousId",
    "setAnonymousId",
  ];
  for (var i = 0; i < methods.length; i++) {
    var method = methods[i];
    rudderanalytics[method] = (function (methodName) {
      return function () {
        rudderanalytics.push(
          [methodName].concat(Array.prototype.slice.call(arguments))
        );
      };
    })(method);
  }

  rudderanalytics.load(<WRITE_KEY>, <DATA_PLANE_URL>);
  rudderanalytics.page();
</script>

<script src="https://cdn.rudderlabs.com/v1.1/rudder-analytics.min.js"></script>
```

<div class="infoBlock">
There is an explicit <code class="inline-code">page</code> call at the end in all the above snippets. This is added to ensure that whenever the SDK loads in a page, a <code class="inline-code">page</code> call is sent. You can either remove this call or modify it with extra <code class="inline-code">page</code> properties as required. You can also add <code class="inline-code">page</code> calls in your application in places not tied directly to page load, for example, virtual page views, page renders on route change, such as in SPAs.
</div>

### Using NPM 

While it is recommended to use the above method to integrate the JavaScript SDK with your website, you can alternatively use the [NPM module](https://www.npmjs.com/package/rudder-sdk-js) to package RudderStack directly into your project.

To install the JavaScript SDK via npm, run the following command:

```bash
npm install rudder-sdk-js --save
```

<div class="warningBlock">
This NPM module is only meant to be used for browser installation. To integrate RudderStack with a Node.js application, refer to the <a href="https://github.com/rudderlabs/rudder-sdk-node">RudderStack Node.js repository</a>.
</div>

Since the NPM module exports the related APIs on an already-defined object combined with the Node.js module caching, you should run the following code snippet **once** and use the exported object throughout your project:

```jsx
import * as rudderanalytics from "rudder-sdk-js";
rudderanalytics.load(<WRITE_KEY>, <DATA_PLANE_URL>);
rudderanalytics.ready(() => {
  console.log("We are all set!!!");
});
export { rudderanalytics };
```

Altrnatively, you can do this with **ES5** using the `require` method, as shown:

```jsx
var rudderanalytics = require("rudder-sdk-js");
rudderanalytics.load(<WRITE_KEY>, <DATA_PLANE_URL>);
rudderanalytics.ready(() => {
  console.log("We are all set!!!");
});
exports.rudderanalytics = rudderanalytics;
```

<div class="infoBlock">
Replace the <Link to="/resources/glossary/#write-key">Write key</Link> and <Link to="/dashboard-guides/overview/#data-plane-url">Data plane URL</Link> in the above snippet with actual values.
</div>

The related APIs exported by the module are:

- `load`                                     
- `ready`                                            
- `identify`
- `alias`
- `page`
- `track`
- `group`
- `reset`
- `getAnonymousId`
- `setAnonymousId`

<div class="infoBlock">
Refer to the sample <a href="https://github.com/rudderlabs/rudder-analytics-angular">Angular</a> and <a href="https://github.com/rudderlabs/rudder-analytics-react">React</a> projects for a detailed walkthrough of the above steps.
</div>

## Step 2: Check ready state

The JavaScript SDK provides `ready` API with a `callback` parameter that triggers when the SDK is done initializing itself and the other third-party native SDK destinations.

An example is shown below:

```javascript
rudderanalytics.ready(() => {
  console.log("All set!");
});
```

## Step 3: Identify users

The <Link to="/event-spec/standard-events/identify/">`identify`</Link> method lets you identify a user and associate them to their actions. It also lets you record any traits about them like their name, email, etc.

A sample `identify` call is shown below:

```javascript
rudderanalytics.identify(
  "1hKOmRA4el9Zt1WSfVJIVo4GRlm",
  {
    email: "alex@example.com"
  },
  {
    page: {
      path: "/best-seller/1",
      referrer: "https://www.google.com/search?q=estore+bestseller",
      search: "estore bestseller",
      title: "The best sellers offered by EStore",
      url: "https://www.estore.com/best-seller/1"
    }
  },
  () => {
    console.log("identify call");
  }
);
```

The JavaScript SDK captures the `userId`, `email` and <Link to="/event-spec/standard-events/common-fields/#contextual-fields">contextual information</Link> from the above snippet.

<div class="infoBlock">
The anonymous visitors are automatically assigned an <code class="inline-code">anonymousId</code>. Refer to the <Link to="/sources/sdks/rudderstack-javascript-sdk/supported-api/#anonymous-id">Anonymous ID</Link> section for more information.
</div>

## Step 4: Track user actions

The <Link to="/event-spec/standard-events/track"><code class="inline-code">track</code></Link> method lets you capture user events along with the properties associated with them.

A sample `track` call is shown below:

```javascript
rudderanalytics.track(
  "Order Completed", {
    revenue: 77.6,
    currency: "USD",
    user_actual_id: 1hKOmRA4el9Zt1WSfVJIVo4GRlm,
  },
  () => {
    console.log("track call");
  }
);
```

The JavaScript SDK captures the `Order Completed` event along with `revenue`, `currency`, and `anonymousId` from the above snippet.

You can use this method to track various success metrics for your website like user signups, item purchases, article bookmarks, and more.
